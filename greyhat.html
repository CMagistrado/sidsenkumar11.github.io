Meeting 5 Problems

This week, we'll be learning about SQL injection. Ever wonder how exactly attackers go about stealing people's usernames, passwords, and data from websites? You hear about companies getting hacked all the time, but how does that happen? Well more often than not, this happens due to a type of attack called SQL injection.




As a quick recap - with any website, we have a front end and a back end. The front end takes care of what you actually see - the colors, the styling, the placement of the words on your screen, etc. This involves languages/tools like HTML, CSS, JavaScript, jQuery, and other front end web-development frameworks. You can see all the front-end code since the front end code is what is sent to your browser to render the page.






On the other hand, many websites have a back end. The back end includes server side languages and scripting languages that actually serve up information for you from a database. The language that a lot of databases use is SQL. SQL is a database query language that, upon request, can read or write to a database. Here is a nice website to learn some of the basics of SQL if you are unfamiliar.

We can use badly coded queries to serve up information to the user that the user of a site might not have been intended to receive by providing special input strings that trick the database. We call this "SQL injection".

Intermediate

There is another encryption scheme called the "One Time Pad". This is quite famous because it is known to be one of the only "perfect" encryptions - that is, when implemented correctly, it has been mathematically proven to be impossible to break.

The way the one time pad works is as follows:

1) Generate a random key at least as long as the plaintext.
2) XOR the plaintext with the key.

It is definitely a very simple algorithm, and can lead to very secure encryption, but when not implemented correctly can lead to vulnerability.
