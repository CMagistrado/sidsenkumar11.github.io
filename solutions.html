Beginner - Bad Hash Algorithm

Take a look at the hashing algorithm to see what is happening to see if we can reverse engineer a ciphertext.
For every character in the plaintext:

1) ord(c) gets the ASCII value of the character. Ex. 'A' becomes 65.

2) This number has an operation done to it; namely //key. We know key to be 3. A google search reveals that the "//" operation in python is floor division (integer division). For example, this means that 81//3 = 82//3 = 83//3 = 27.

Let's now look at the first number in the ciphertext: 27.
27 is the quotient after the ascii value of the first letter in the plaintext was floor divided by 3. What are the possible numbers that can result in 27?

Well, as I showed in 2), all three of 81, 82, and 83 yield 27 when we floor divide by 3!
So, for 27, we know that the plaintext ascii value is either 81, 82, or 83. These correspond to the ASCII characters 'Q', 'R', and 'S' (you can use an ASCII table to verify this).

Intermediate - SHA-256 Brute Force

Hash: f82fae9becc67392b1bdf9a9ba753d8c7838f0aa0e806455625875bdf0992122
Password format: [car brand][0-9][0-9][0-9][punctuation mark]
Example: TOYotA111.

1) The SHA-256 is a fairly secure hash so the only we're going to be able to reverse this is by a bruteforce dictionary attack. Since we know the format of the password, let's create a few dictionaries and save them as text files.

2) We're basically going to append every combination of words from the dictionaries, hash them, and see if the hash matches. Ex. Try toyota111. , then try toyota111!, then try toyota111?, .... honda111. , honda111! and so on.

If a hash matches the one given, we found our password. Note that there is no need for you to learn exactly how the SHA-256 hashes the words; people have already written many hash functions for you to use.

A few friendly notes:
1) Remember to try every combination of capital and lowercase letters in the car word. So the password could have had "TOyota" or "tOYotA" or any combination of capitals and lowercase letters.
2) A few people had the right script to come up with permuations of words but could not find the correct password. Remember to add as many words as possible to your dictionary! In my dictionary I did not put that many but you should.
3) Of course, the more words you have in your dictionaries, the longer the script will take to run. For me, the script took ~30 seconds with an i7 processor.
